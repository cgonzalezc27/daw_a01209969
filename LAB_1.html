Descripción de conceptos

URL

URL son las siglas para Uniform Resource Locator en inglés. El URL es una especie de dirección para los diferentes recursos que se pueden encontrar en internet. Estas direcciones están estandarizadas y divididas en dos secciones separadas con dos puntos; el protocolo y el nombre del recurso. El recurso, por su parte, está compuesto por el nombre del servidor en la red donde se encuentra alojado el recurso, y la ruta donde está el recurso (separados por “/”). [1]

MIME

Es un estándar cuyo objetivo fue ampliar las capacidades (antes limitadas) del correo electrónico. En inglés es conocido como Multi-purpose Internet Mail Extensions y permite que el usario inserte archivos a los correos. [2] [3]

Métodos y códigos de errores http

Los métodos utilizados en http permiten indicar la acción que se pretende realizar para el recurso en cuestión. Los métodos más comunes son GET, HEAD, POST, PUT, DELETE, CONNECT, OPTIONS, TRACE y PATCH.

Los errores de cliente empiezan por el número 4 y algunos de los más comunes son 401, 403 y 404.

Los errores de servidos empiezan por 5 y un ejemplo es el 502 Bad Gateway. Es importante mencionar que se busca que este tipo de errores no son deseables en una aplicación. [4] [5]

XML

XML es conocido como Extensive Markup Language. El objetivo principal de este lenguaje es el transporte o el alamcenamiento de datos. Contrasta con el lenguaje HTML en el sentido de que este último se centra en la presentación de dichos datos. De esta manera, al momento en que se hace algún cambio en los datos que se presentan, el HTML no debería pro qué cambiar, solo el XML. Es importante mencionar que el XML no tiene tags definidos, el autor del archivo debe definir los tags. [6]

XHTML

Se define como Extensible HyperText Markup Language. Es muy similar a HTML, solo que su principal diferencia es que el XHTML es un lenguaje un poco más estricto, pues combina algunas funcionalidades y características del XML al HTML. Esto, con el fin de evitar que se den situaciones donde por no seguir algunas del las reglas del HTML, los exploradores no puedan interpretar el código. [7]

HTML5

Es la versión más actualizada de HTML que engloba diferentes herramientas y tecnologías más avanzadas que permite la generación de aplicaciones más diversas. Se dice que HTML5 tiene dos partes, la nueva versión de HTML y el 5 que representa el acceso a las nuevas tecnologías mencionadas anteriorment. [8]

JS

Javascript es un lenguaje orientado a objetos, multi-paradigma. Su principal objetivo es programar el comportamiento de las aplicaciones web. De esta manera, las aplicaciones pueden tener una interactividad dinámica, pues el uso de este lenguaje puede ser incorporado a la creación de aplicaciones HTML. [9] [10]

php

Se refiere a Hypertext Preprocessor y es un lenguaje muy usado que permite que los scripts sean ejecutados en el servidor y el resultado sea llevado al explorador como HTML. Además, permite al desarrollador evitar utilizar muchos comandos que normalmente serías utilizados para mostral HTML, reemplazándolos con funciones PHP. [11] [12]

jsp

Significa JavaServer Pages que es una tecnología la cual permite al desarrollador crear aplicaciones dinámicas de manera simple y rápida. Funciona de manera similar a PHP solo que es de Java. [13]

Ajax

AJAX no es un lenguaje de programación, sino que se conoce como una “técnica” que permita acceder a servidores desde una página web. Esto quiere decir que el contenido de una página web puede ser actualizado sin que la página tenga que volverse a cargar. Las siglas significan Asynchronous JavaScrip And XML. [14]

Preguntas:

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet se compone por una serie de interconexiones tanto físicas como digitales, conectando a los computadores y demás dispositivos electrónicos en una sola red. Mientras tanto, la WWW no es la conexión, sino que es el software que se transporta través del internet en forma, por lo general, de contenido. [15]

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: recuperar datos.
  • HEAD: similar al GET, solo que aquí solo se recupera el encabezado de la respuesta.
  • POST: manda información al servidor.
  • PUT: reemplaza el contenido del recurso con el contenido actualizado.
  • PATCH: se aplica para modificaciones parciales del recurso.
  • DELETE: elimina un recurso específico.

[4]

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST, pues se está enviando el formulario.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET, pues el navegador trae contenido.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que no hay ningún error, el método se implementó con éxito. [5]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No necesariamente, pues es un error que se da por responsabilidad del usuario. [5]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, pues significa que la aplicación tiene un error por lo que no se está pudiendo obtener un resultado exitoso al implementar el método. [5]

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Quiere decir que el atributo ha sido reemplazado por otro con una mejor funcionalidad o mayor flexibilidad. Ejemplos [16]:

  • acronym
  • applet
  • basefont
  • big
  • center
¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Las principales diferencias pueden resumirse como las siguientes: en HTML4 el desarrollador debía definir el DOCTYPE a partir de un repertorio de varios tipos mientras que en HTML5, solo se hace con un tipo de documento; las definición del conjunto de caracteres es mucho más sencilla HTML5; HTML5 introdujo nuevos elementos que permiten más flexibilidad y nuevas funcionalidades al desarrollador; en HTML5 se le agregaron algunos atributos extra a tags de HTML4; también la nueva versión introdujo formularios nuevos que permitan al usuario un llenado más sencillo y dinámico; y, por último, se integraron APIs. [17]

¿Qué componentes de estructura y estilo tiene una tabla?

Estructura:

  • Align: Específica la alineación del texto frente al resto del contenido.
  • Cellpadding: Sirve para determinar el espacio entre el borde y el contenido de una celda.
  • Cellspacing: Tiene una funcionalidad similar al tag anterior.
  • Width: Define el ancho de la tabla.

Estilo:

  • Bgcolor: Da color al fondo de la tabla.
  • Border: Especifica el grosor del borde de la tabla.

Es importante resaltar que muchos de estos atributos ya no son utilizados, pues las tablas se deben generar en CSS donde algunos ejemplos de los atributos pueden ser los siguientes:

Estructura:

  • Width: Define el ancho de la tabla.
  • Height: Define el alto de la tabla.
  • Text-align: Especifica la alineación del texto dentro de la tabla.
  • Vertical-align: Especifica la alineación vertical del texto.
  • Padding: Establece el espacio entre la tabla y sus elementos.

Estilo:

  • Border: Define el estilo del borde.
  • Border-collapse: Específica si los bordes deben ser una sola línea.
  • Border-bottom: Añade solo bordes a la parte inferior de los textos.
  • Background-color: Definir el color del fondo.
